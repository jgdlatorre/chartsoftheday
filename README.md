# Charts of the Day (COTD) · CandleEye v0.9.4

Visual chart-reading skill test. Five 10-second decisions per day, weekly streak unlocks rewards pool.

## Modes

The server has two modes:

1. **Pool mode** — reads `sections_pool.json` (~1-2 MB). Deploy-ready, no CSVs needed. Default when `sections_pool.json` is present.
2. **CSV mode** — reads raw `data_1h/`, `data_4h/`, `data_1d/` CSVs (~150 MB). Local dev only. Fallback if no pool file found.

## Local dev

Requires the full CSVs from `data_prep.py`.

```bash
pip install -r requirements.txt
python data_prep.py         # one-time, generates CSVs from Binance parquets
python server.py            # serves on localhost:5050
```

Tests available on iPhone via same-WiFi IP printed at startup.

## Build the pool for deploy

Once CSVs exist locally, extract a lightweight section pool:

```bash
python extract_sections.py          # 100 sections, ~1-2 MB
python extract_sections.py --n 200  # more if you want
```

This writes `sections_pool.json` in the repo root. Commit this file — it's what production will serve.

## Access token (password)

Set `ACCESS_TOKEN` env var to gate the app behind a single shared password.

- First visit: user sees a small password form at `/`. Submit → cookie set, full access.
- Alt: append `?k=<token>` to any URL → cookie set automatically on response.

If `ACCESS_TOKEN` is empty or unset, auth is disabled entirely (useful for local dev).

## Deploy to Railway

1. Push the repo to GitHub (include `sections_pool.json`, exclude `data_*` and `sessions/`).
2. In Railway: New Project → Deploy from GitHub repo.
3. Railway auto-detects Python. It installs from `requirements.txt` and runs `Procfile`.
4. In Railway settings → Variables, add:
   - `ACCESS_TOKEN=<your-password>`
5. Custom domain (optional): add a domain under Settings → Networking.

The Procfile runs gunicorn with 2 workers, suitable for the free tier.

## File manifest for deploy

Include in the repo:
- `server.py`
- `static/index.html`
- `sections_pool.json` (generated by `extract_sections.py`)
- `requirements.txt`
- `Procfile`
- `.gitignore`

Exclude (not needed in production):
- `data_1h/`, `data_4h/`, `data_1d/` (raw CSVs)
- `data_prep.py`, `extract_sections.py` (build-time scripts)
- `sessions/` (local analytics dumps)
- `__pycache__/`

## Architecture

- `server.py` — Flask app. Auth middleware, endpoints `/daily/<n>`, `/decide`, `/session`, `/meta`. Pool loader.
- `extract_sections.py` — one-time script: picks N random (symbol, decision_ts) tuples from CSVs and serializes them to JSON.
- `static/index.html` — single-file SPA. Dark UI, i18n ES/EN/ZH. No external deps.
- `sections_pool.json` — generated artifact, committed to repo, read at boot.
